Document Landing Page

Experimental: This component is currently experimental because more research is needed to validate it.

Demo document landing page

When to use

Content published in a document such as a PDF, Word document or Excel spreadsheet should have an associated web page (document page). The document page will give information about the document and will link to it.

Use a document landing page to help the user find the document that matches their needs as quickly as possible.

Users need to know whether they should open the document and if the information within meets their needs.

How to use

Document page template

Work from a document page template, provided by the content design team.

This template is to be completed by the content design team.

Title

The document title is a <h1> tag, it should be included at the start of the content region.

  • Title does not have to reflect the official document title
  • Keep it short, under 65 characters if possible, user friendly and optimised for search
  • Frontload the title – get the most important words at the front
  • You can mention the document title etc in the summary below or details section if you think users will search for that term

Summary

The summary is a sentence using a <p> tag, it should be included directly below the <h1> title.

  • No heading
  • Explain the main purpose of the document
  • Up to 160 characters and end with a full stop

Version

Document version details are to be included directly below the summary. Use the class of .document with a <dl> list. The details to include should be provided in the document page template.

Title Detail Mandatory or optional
Applies to If the document applies to Great Britain (GB) (England, Scotland and Wales) leave this section blank. If it does not apply to GB specify which nation(s) it applies to. Mandatory
Published, Last updated
  • Content designer – request entries if the user needs to know the date(s).
  • Publisher – if required, enter date(s) document originally published/last updated.
 Optional

Document

The document section should be given a <h2> heading of 'document' and is included directly below the version details.

A box component with the additional class of .document, is used to emphasise a document on a page.

  • Use the official title
  • This is the link to the document
  • Insert live link or link text and address/URL separately

A <dl> tag with the class of .document is used to include:

  • Reference (optional): Use if document has a reference
  • File type (mandatory): PDF, Microsoft Word, Microsoft Excel etc, file size in MB or KB

Code example

<div class="box document">
 <h3><a href="#">Document title</a></h3> 
 <dl class="document">
  <dt>Reference:</dt>
  <dd>Version 1</dd>
  <dt>Document type:</dt>
  <dd>PDF, 600KB</dd> 
 </dl> 
</div>

Document title

Reference:
Version 1
Document type:
PDF, 600KB

Details

The details section should have a heading of 'Details' and always be the last section of content, it is mandatory to include.

  • Provide a description of the publication in plain, neutral language, to reassure the user that it is (or is not) what they’re looking for
  • Include what the publication is about, who it’s for and its purpose
  • Do not repeat the summary text

Related content

  • Links to supporting content or content a user may need next
  • No more than 4 ideally. Include essential links in the details section
  • Insert live link or link text and address/URL separately

Explore this topic

  • Single link to topic area - for example, for a document on ladders the link would go to the topic area ‘Work at height’
  • Insert live link or link text and address/URL separately
