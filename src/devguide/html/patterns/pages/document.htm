Document Landing Page

Experimental: This component is currently experimental because more research is needed to validate it.

Demo document landing page

When to use

This guidance does not apply to publications from HSE Books.

Use this page format for documents that are free to download when publishing a new document or updating an existing document.

It should be short and straightforward. Its purpose is to help the user find the document that matches their needs as quickly as possible.

Users need to know whether they should open the document and if the information within meets their needs.

How to use

Title

The title does not have to reflect the official document title. Keep it short and search- and user- friendly.

Frontload the title – get all the most important words at the front of the sentence.

You can mention the document title etc in the summary or page copy if you think users will search for that term.

Summary

Give a short summary (up to 160 characters) and end with a full stop.

Last updated

Under the summary, consider inserting the date the document was last updated; for example, ‘Last updated 6 January 2021’.

Documents

A box component with the style class of .document can be used to emphasise a document on a page.

Each document should have a title using a <h3> element as a link to the file.

Document details should be included using a description list.

Use the official title of the document and include the file type and memory size.

Body copy (Details)

The body of the page provides a description of the publication in plain, neutral language, to reassure the user that it is (or is not) what they’re looking for. Include what the publication is about and its purpose.

Include any further information that will help them complete their task where it’s not covered in the document; for example, related links.
