Document Landing Page

Experimental: This component is currently experimental because more research is needed to validate it.

Demo document landing page

When to use

Documents such as PDFs, Microsoft Word docs and Excel spreadsheets published on the HSE website should have a webpage.

There may be exceptions where it is not practical.

Use a document landing page to help the user find the document that matches their needs as quickly as possible.

Users need to know whether they should open the document and if the information within meets their needs.

How to use

Document page template

Work from a document page template, provided by the content design team. This template is to be completed by the content owner or content design team.

Title

The document title is a <h1> tag, it should be included at the start of the content region.

The title does not have to reflect the official document title. Keep it short and search-friendly.

Frontload the title – get all the most important words at the front of the sentence.

You can mention the document title etc in the summary or page copy if you think users will search for that term.

Summary

The summary is a sentence using a <p> tag, it should be included directly below the <h1> title.

Give a short summary (up to 160 characters) and end with a full stop.

Version

Document version details are to be included directly below the summary. Use the class of .document with a <dl> list. The details to include should be provided in the document page template.

Document

The document section should be given a <h2> heading of document or documents and is included directly below the version details.

A box component with the additional class of .document, is used to emphasise a document on a page.

  • Each document should have a heading using a <h3> element as a link to the file
  • Document details should be included using a <dl> list with the class .document
  • Use the official heading of the document and include the file type and memory size
  • Multiple formats of the document can be listed in this section, use a box componenet for each individual format of the document

Details

The details section should always be the last section of content.

The body of the page provides a description of the publication in plain, neutral language, to reassure the user that it is (or is not) what they’re looking for. Include what the publication is about and its purpose.

Include any further information that will help them complete their task where it’s not covered in the document; for example, related links.
