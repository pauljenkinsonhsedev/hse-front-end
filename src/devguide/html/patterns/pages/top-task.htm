Top task

When to use

A top task page should be used when there is a group of pages that are related and can be sequenced to make it easier for the user (e.g. An employer) to understand and meet their Health and Safety requirements.

How to use

To use the top task page design you will need to set the values of the layout and columnLayout template parameters to the following:


<!-- InstanceParam name="layout" type="text" value="betatwocol" -->
<!-- InstanceParam name="columnLayout" type="number" value="2" -->

Top task elements

1. Pagination (ordered page list)

Top task pages include navigation in the form of an ordered list at the top of each page, immediately below the <h1> title in an editable region named multistepNav. The list should contain the core pages of the top task.

The pagination page list is a ol element with the class of .multistep, the list items contain every page of the top task and includes an in-page link to jump to the article section of the the top task page.

It needs to be included in every page of the top task, it's easier to manage this using a library item.


<ol class="multistep">
<li><a id="step1" href="index.htm#article">Overview</a></li>
<li><a id="step2" href="how-to-write-your-policy.htm#article">How to write your policy</a></li>
<li><a id="step3" href="the-law.htm#article">The law</a></li>
</ol>

2. Article

The article region contains the main content of the page, all content components can be used in this region.

3. Pagination (previous, next)

Previous and next links are included on each top task page, the componenet pattern is included inside the article region as the last element with no content or components below it.

The first page of the top task will only need the next link and the last page will only need the previous link.

4. Complementary content

The complementary content is always included in the asideBottom editable content region and should relate to the page content.

The resources box should always come before the related content box on a top task page.
