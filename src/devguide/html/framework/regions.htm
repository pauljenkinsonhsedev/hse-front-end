Editable template regions

Editable template regions are areas of a template-based page that can be edited. Some regions are intended to contain a specific componenet.

Editable region code

Editable regions are clearly visible when using design view in Dreamweaver, if you are using code view you can find editable regions by looking for the editable region HTML commented code.

Template regions

Template region names display in design view and mark the editable regions. The table below lists all editable regions in the website template, some regions are conditional and will only display when a specific parameter value has been selected.

Document title and meta elements

Document title

The HTML Title element defines the document's title that is shown in a browser's title bar or a page's tab, it only contains text.

  • Every page should have a document title
  • The document title should have similar text to the document H1 tag

Meta description

The meta description is an HTML attribute that provides a brief summary of a web page. Search engines such as Google often display the meta description in search results.

Page notification

The page notification is designed to contain a short message and a link to further information. It displays below the main menu and above the page breadcrumbs.

It can be hidden by the user when the click the link text 'hide message'.

The nofication message can be applied using template parameters.

When to use

Page notifications should be used to tell the user about something important they need to know about, that is related to the page content.

If you need to display a notification on every page on hse.gov.uk, use the notification banner template component.

Breadcrumbs

Breadcrumbs on HSE.GOV.UK display a hierarchy of the current page in relation to the the site's structure, from homepage (top level) to current page.

  • The first list item in the breadcrumbs should always be HSE.GOV.UK homepage
  • The first list item should also have the class home assigned to it
  • The last list item should be the current page
  • The current page (last list item) has no link

Accessibility

The aria state: aria-current="page" is applied with JS to the last list item of the breadcrumbs to indicate the current page in the breadcrumbs.

Beta survey

Content region for description and link to beta survey.

Currently this is applied with a library item and is the same on all pages.

Secondary navigation

The secondary navigation region is available when the page parameter column layout value is set to 2.

The secondary navigation is a menu which is an unordered list that can contain nested lists

  • Always apply a class of 'first' to the first list item in the menu
  • Keep list item text to one line
  • Do not wrap list item text
  • The menu structure should reflect the site architecture (folder structure)
  • The parent li item of any nested list should contain a # as it's href value
  • The first li item of a nested list should only have the text 'Overview'

Accessiblity

As the text 'Overview' can be used multiple times in the menu and link to different destinations, JS is used to automatically append the parent list item text to it which is visually hidden from screen.

Content

The content area consists of content that is directly related to or expands upon the central topic of the page.

Layout, Components and typography can be used within this region to create content.
