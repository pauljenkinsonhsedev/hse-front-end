User satisfaction

Each webpage on HSE.GOV.UK has the option to add user feedback at the top of the webpage footer, this is optional and can be displayed using template parameters.

Is this page useful

When to use

User feedback is currently applied to all pages and is an important part of measuring user satisfaction on HSE.GOV.UK.

How to use

Set the feedback param value to true and run a template update on the page.

If you are applying the user feedback component to multiple pages, you can run a find and replace, followed by a template update.

<!-- TemplateParam name="feedback" type="boolean" value="true" -->
<!-- TemplateParam name="feedbackClass" type="text" value="" -->

Survey

The survey section is an option that can be used in conjunction with the default user feedback survey. When the survey is enabled, if a user selects yes or no they will then be prompted to complete a survey.

How to add survey

To include a survey you will need to add a value to the feedbackClass parameter, as this suvery is experimental there is currently only one value which is 'feedback-beta'.

Survey libray item

The survey should be contained in a libray item and applied to all pages in site or specific pages you want the survey to display. You must include the survey in the editable region named 'survey', the library item does not require a specific name.

<!-- InstanceBeginEditable name="survey" -->
<!-- #BeginLibraryItem "/Library/feedback-survey.lbi" -->

<!-- #EndLibraryItem -->
<!-- InstanceEndEditable -->

The survey library item contains HTML content which requires a specicic structure and style classes, you can use the survey library item example to build your own.

Report a problem with this page

The 'report a problem with this page' section is intended for users to report problematic pages on hse.gov.uk, this information is used by the web and design team to investigate potential issues on hse.gov.uk website.

When a user selects the 'Report a problem with this page' button, a HTML form is inserted with JS to provide the user input fields to report a problem.

The two input fields are:

Field name Description Input type
What were you doing? User describes what they were doing at the time a problem occured on the page, e.g 'I clicked on a link to a PDF' Text
What went wrong? User describes the problem, e.g. 'The PDF file did not open' Text

When the user submits the form it sends an email to hse.online@hse.gov.uk, the email includes the user input fields and a hidden field that contains the URL of the page the user was on when they completed the form.
