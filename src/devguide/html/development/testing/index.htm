Framework testing

Testing HSE.GOV.UK website framework for usability and technical issues.

Testing goals

  • Website is easy to use for anyone who needs to use it, regardless of the device theyâ€™re using
  • Website is stable, secure and works quickly, regardless of how many people need to use it
  • Website can be iterated quickly to meet changes to user needs or the political environment

Testing workflow

Phase 1 - Local testing

  • Apply new template to test pages
  • Load test pages with framework assets in local environment
  • Exploratory testing
  • Test responsive view in chrome developer tools
  • Check console for JS errors
  • Test pages in all supported browsers

Phase 2 - Mobile testing

  • Upload test pages and framework assets to external testbed: https://beta.hse.gov.uk/testbed/
  • Mobile device testing

Phase 3 - Performance and accessibility

  • Performance
    • Chrome DevTools: Inspect Network Activity
    • Chrome DevTools: Analyse Runtime performance
    • Run lighthouse performance report in chrome
    • Check reports against previous framework
  • Accessibility
    • Run lighthouse accessibility report in chrome
    • Use siteimprove accessibility checker (chrome extension) to check for new errors and issues
    • Keyboard-only navigation test

Reports

Reports should be compared to latest stable version of website framework, any significant negative impact on performance should be investigated further and resolved before deploying framework release.

Accessibility

Accessibility Reports should be compared to latest stable version of website framework, if framework release includes specific siteimprove fixes, they can be checked using the siteimprove accessibility checker.
