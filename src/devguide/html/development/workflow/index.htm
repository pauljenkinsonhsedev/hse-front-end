Overview

We use git for version control. For our workflow we use Gitflow, an extension of Git which has been designed to simplify development and release management.

The workflow cycle

The workflow cycle is very simple. Any new features created and ongoing maintenance will all take place during next release. Once the work has been tested and a release date has been set the release can go into production and be made live.

After that in the event there are fixes to be made (which should be minor) fixes can be incorporated in to both development and production cycles.

More on the workflow cycle for development can be explored in how we use Gitflow.

Workflow steps

  • New features and general maintenance
  • Testing
  • Release
  • Fixes

Peer reviewing

During the inital development phase of a release it is good practice to peer review with your colleagues. One way we can do this is to use "Pull requests" or "PR" as you might hear it reffered to from time to time.
